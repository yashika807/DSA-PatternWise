Prefix-Sum · Monotonic Deque · LeetCode 862 · Hard

Shortest Subarray Sum ≥ K

Build the prefix sum array upfront. Then walk checkpoints left → right, keeping a deque of strictly increasing checkpoints — the only ones still "promising." Whenever the newest checkpoint clears the gap to the front (prefix[j] - prefix[front] ≥ k), record that window and discard the front — it can only produce longer windows from here on. Dominated older checkpoints get trimmed from the back too.

Time  O(n) Space  O(n) Pattern  Prefix Sum + Monotonic Deque
